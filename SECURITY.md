# Security Policy

## Supported versions

This repository is still early-stage, so security fixes are applied on the latest default branch (`main`).

## Reporting a vulnerability

Please do **not** open a public GitHub issue for suspected security problems.

Instead:

1. Contact the maintainer privately first.
2. Include a short description of the issue.
3. Include reproduction steps, affected files, and impact.
4. If relevant, mention whether the issue affects:
   - URL fetching
   - local file ingestion
   - text extraction
   - packaged skill distribution

## What to include

Helpful details:

- example input type
- exact command used
- observed behavior
- expected behavior
- any logs or stack traces with secrets removed

## Disclosure preference

Please allow time for validation and a fix before public disclosure.
